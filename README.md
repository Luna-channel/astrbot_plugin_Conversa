# Conversa · AI 主动续聊插件 for AstrBot

> **作者**：柯尔 (Luna-channel)
> **版本**：v3.0.0
> **仓库**：<https://github.com/Luna-channel/astrbot_plugin_Conversa>

Conversa 让 AI 像真人一样主动找你聊天 —— 在对话沉寂一段时间后自然地重新开口，或者在你们正聊着的时候概率性地追加一句话，营造更真实的对话体验。

---

## v3.0 更新亮点

**主动回复全面接入 AstrBot 官方 Agent Pipeline。**

v2 及之前的版本直接裸调 LLM，绕过了框架的事件处理链。v3 改为使用框架的 `CronMessageEvent` + `build_main_agent()` 走完整 Agent 流程，这意味着：

- 主动回复现在**支持 Agent 工具调用**（搜索、代码执行等）
- 其他插件的 hook **正常触发**
- 人格 / System Prompt 由**框架自动注入**，不再需要手动获取
- 对话历史由**框架自动管理**

旧版本框架自动降级兼容，无需用户操作。

完整更新日志见 [CHANGELOG.md](CHANGELOG.md)。

> ## ⚠️ 旧版用户：请迁移提醒数据
>
> 如果你从 v2.x 升级，且使用过 `/conversa remind` 设置过提醒，请在 AstrBot 聊天中执行：
>
> ```
> /conversa migrate-reminders
> ```
>
> 此指令会将你的旧提醒迁移到 AstrBot 原生定时任务系统。**旧数据保留不删除**，可继续通过 `/conversa remind` 管理。可重复执行，已迁移的会自动跳过。
>
> 推荐后续直接对 AI 说「提醒我...」使用原生定时提醒，效果更好且支持自然语言。

## 核心功能

### 沉寂问候

与用户最后一次聊天结束后，经过一段可配置的沉寂时间，AI 会主动发起一条问候消息。每轮对话后必定触发一次。

- 触发时间 = 基准时间 +/- 随机波动（最低 30 分钟）
- 用户可通过 `/conversa set after <小时>` 设置个人专属触发时间
- 支持多个提示词模板随机选择

### 私聊对话增强

AI 正常回复用户后，短时间内有概率主动追加一条消息，像真人聊天一样自然接话，提升活人感。仅私聊生效。

- 可配置触发概率、等待时间范围
- 指数递减防刷屏：连续触发时概率按系数衰减
- 用户在等待期间发新消息时自动取消

### Agent 订阅

支持三种订阅方式：
- `manual` — 用户手动 `/conversa watch`
- `auto` — 用户首次发消息自动订阅
- `agent` — AI 通过 Agent 工具调用自动管理订阅

### 其他特性

- **免打扰时段**：全局或用户专属，支持跨天（如 23:00-07:00）
- **自动退订 & 重新激活**：不活跃用户自动退订，再次发消息自动恢复
- **人格继承**：完整使用 AstrBot 的人格系统，可通过高级设置强制覆盖

## 安装

在 AstrBot WebUI → 插件市场中搜索 **Conversa** 安装，或手动将插件放入：

```
AstrBot/data/plugins/astrbot_plugin_conversa/
```

重启 AstrBot 后在 WebUI 中启用即可。

---

## 快速上手

1. **订阅会话**

   在聊天中发送：
   ```
   /conversa watch
   ```
   或在 WebUI 插件配置的「订阅用户列表」中添加会话 ID。

2. **等待触发**

   默认沉寂 1200 分钟（20 小时）后触发问候。可以调短试试效果：
   ```
   /conversa set after 1
   ```
   （设置为 1 小时后触发）

3. **（可选）设置免打扰**
   ```
   /conversa set quiet 23:00-07:00
   ```

## 指令一览

| 指令 | 说明 |
|---|---|
| `/conversa help` | 显示帮助 |
| `/conversa debug` | 显示调试信息 |
| `/conversa on` / `off` | （管理员）启用/停用插件 |
| `/conversa watch` / `unwatch` | 订阅/退订当前会话 |
| `/conversa set after <小时>` | 设置个人沉寂问候时间（最低 0.5） |
| `/conversa set quiet <HH:MM-HH:MM>` | 设置个人免打扰时段 |
| `/conversa set quiet <...> global` | （管理员）设置全局免打扰 |
| `/conversa remind add <HH:MM> <内容>` | 添加每日提醒 |
| `/conversa remind add <YYYY-MM-DD HH:MM> <内容>` | 添加一次性提醒 |
| `/conversa remind list` | 查看提醒列表 |
| `/conversa remind del <序号>` | 删除提醒 |

## 配置说明

所有配置在 AstrBot WebUI → 插件配置中修改。

### 基础设置

| 配置项 | 默认值 | 说明 |
|---|---|---|
| 订阅方式 | `manual` | `auto` / `manual` / `agent` |
| 免打扰时段 | 空 | 格式 `HH:MM-HH:MM`，支持跨天 |
| 自动退订天数 | 3 | 用户连续无回复天数后自动退订，0 禁用 |
| 回复间隔（秒） | 20 | 多个主动回复之间的间隔 |
| 时区 | 系统时区 | IANA 格式，如 `Asia/Shanghai` |

### 沉寂问候

| 配置项 | 默认值 | 说明 |
|---|---|---|
| 启用 | 开 | 总开关 |
| 沉寂基准时间（分钟） | 1200 | 最后一次聊天后多久触发 |
| 随机波动（分钟） | 30 | 触发时间的随机浮动范围 |
| 问候提示词模板 | 内置 2 个 | 支持多个模板随机选择 |

### 私聊对话增强

| 配置项 | 默认值 | 说明 |
|---|---|---|
| 启用 | 关 | 总开关 |
| 触发概率（%） | 20 | 每次 AI 回复后的触发概率 |
| 最短/最长等待（秒） | 30 / 600 | 触发后的等待时间范围 |
| 递减系数 | 0.1 | 连续触发时概率的衰减倍率 |

### 高级设置

| 配置项 | 默认值 | 说明 |
|---|---|---|
| 固定 Provider | 空 | 指定主动回复使用的 LLM Provider |
| 人格覆盖 | 空 | 强制覆盖框架人格，仅对 Conversa 生效 |
| 历史条数（降级模式） | 50 | 仅旧版框架降级模式下生效 |

### 提示词占位符

所有提示词模板支持以下占位符：

| 占位符 | 含义 |
|---|---|
| `{now}` | 当前时间 |
| `{time_since_last_chat}` | 距离上次聊天过了多久（如"3小时20分钟"） |
| `{last_user}` | 用户最后一条消息（截取前 200 字） |
| `{last_ai}` | AI 最后一条回复（截取前 200 字） |
| `{umo}` | 会话 ID |

## 数据存储

插件业务数据存放在 `AstrBot/data/plugin_data/astrbot_plugin_conversa/`：

| 文件 | 内容 |
|---|---|
| `user_data.json` | 用户订阅信息、提醒事项 |
| `session_data.json` | 会话运行时状态（时间戳、idle 计时等） |

对话历史由 AstrBot 框架统一管理，插件不单独存储。

## 故障排查

**没有收到主动消息？**
- `/conversa debug` 确认订阅状态和插件是否启用
- 检查是否处于免打扰时段
- 确认沉寂时间是否已到（默认 1200 分钟 = 20 小时）
- 确认时区设置是否正确

**消息内容不理想？**
- 在 WebUI 中修改提示词模板
- 检查人格设置是否生效
